## Context

墨渊前端基于 React 19 + Tailwind CSS v4 + Vite，当前设计系统定义于 `src/index.css`，采用「科技冷调」风格。共有 14 个页面、20+ 组件、完整的 light/dark 主题系统。

**施主圣谕：大争之世，当勇往直前！除功能外，皆可改变。**

## Goals / Non-Goals

**Goals:**
- 彻底重构布局架构：顶部导航 → 道观经络式侧边栏
- 彻底重构组件体系：废弃 `.taste-*`，创建全新 `.dao-*` 组件系统
- 彻底重构视觉范式：从现代科技风 → 赛博道观美学
- 彻底重构页面组织：按「道藏、丹室、玄台」三大模块重组

**Non-Goals:**
- 不改变任何功能行为或 API 接口
- 不修改后端代码
- 不修改 `moyuan-landing-final.html`（已独立完成）

## Decisions

**Decision 1: 道观经络式侧边栏布局**
将当前顶部导航改为左侧固定侧边栏，以八卦方位划分三大模块：
- **乾位（西北）**：道藏（知识库、笔记、标签、分类）
- **坤位（西南）**：丹室（工作区、统计、备份）
- **中位（中央）**：玄台（设置、回收站）
→ 优势：视觉层次分明，符合道观布局逻辑；劣势：首次加载需适应新布局

**Decision 2: 废弃 taste-*，全新创建 dao-* 组件体系**
完全抛弃现有的 `.taste-*` 命名空间，从零开始创建 `.dao-*` 组件：
- `dao-card`：灵气卡片（墨色背景、翠玉边框、毛玻璃效果）
- `dao-btn`：道符按钮（翠玉/丹金配色、hover 气晕）
- `dao-input`：符纹输入框（玄青背景、focus 灵气光晕）
- `dao-dialog`：结界对话框（八卦边框、丹金分隔线）
- `dao-badge`：符印徽章（深色背景、翠玉文字）
→ 优势：彻底摆脱旧风格束缚；劣势：需完全重写所有组件引用

**Decision 3: 三大模块架构重组**
按道家宇宙观重组页面路由：
- **道藏**：知识库、笔记、标签、分类、收藏、合集（知识之道）
- **丹室**：工作区、统计、备份（修炼之道）
- **玄台**：设置、回收站（调控之道）
→ 优势：逻辑清晰，符合道家思想；劣势：路由结构需调整

**Decision 4: 四色主调 + 金色点缀**
以落地页验证的四色为基础：
- `--ink: #08080c`（墨色 - 主背景）
- `--void: #0d1117`（玄青色 - 层次背景/侧边栏）
- `--jade: #10b981`（翠玉色 - accent/交互）
- `--gold: #c9a84c`（丹金色 - 点缀/重要元素）
→ 优势：色彩系统简洁有力，视觉冲击力强

**Decision 5: 全局粒子气机背景 + 八卦结界装饰**
在 App 根级别注入：
1. Canvas 粒子系统（翠玉色粒子，模拟气机流散）
2. SVG 八卦结界装饰（页面四角、卡片边框）
3. 墨绿网格背景（60px 网格线，电子经络意象）
→ 优势：全站氛围统一，赛博道士感强烈；劣势：需注意性能优化

**Decision 6: 道家术语全面替换**
所有文案改用道家术语：
- 搜索 → 问玄
- 知识库 → 道藏
- 笔记 → 墨宝
- 标签 → 符印
- 分类 → 坤舆
- 收藏 → 珍藏
- 合集 → 藏经
- 统计 → 卦象
- 工作区 → 丹室
- 备份 → 封魔
- 回收站 → 归墟
- 设置 → 玄台

## Risks / Trade-offs

- **[Risk]** 用户首次使用需适应新布局 → **Mitigation**：布局变化显著但逻辑清晰，侧边栏导航直观
- **[Risk]** 组件重构工作量大 → **Mitigation**：按优先级分批实施，先完成核心组件
- **[Risk]** 粒子系统影响性能 → **Mitigation**：Canvas 仅在深色模式启用，根据设备性能自适应粒子数量
- **[Risk]** 术语替换影响理解 → **Mitigation**：hover 显示 tooltip 说明原功能名

## Migration Plan

分四阶段：
1. **布局架构重构**：创建侧边栏组件、重写 App.tsx 布局、重组路由（2-3天）
2. **组件体系重构**：创建全新 `.dao-*` 组件，替换所有旧组件引用（3-4天）
3. **页面逐一重构**：按三大模块重构 14 个页面（4-6天）
4. **动效系统集成**：粒子气机、八卦结界、灵气感应（2-3天）

无数据库迁移，无 API 变更，可随时回滚。
