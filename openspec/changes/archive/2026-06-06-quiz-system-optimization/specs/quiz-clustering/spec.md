## ADDED Requirements

### Requirement: 关键词语义聚合出题
系统 SHALL（未来阶段）支持用户传入关键词，在指定范围内向量检索 → 自动聚类 → 从同一聚类簇抽取源块出题，实现同知识点集中刷题（专项练习模式）。

#### Scenario: 关键词聚合
- **WHEN** 用户传入关键词如"傅里叶变换"
- **THEN** 系统 SHALL 向量检索范围内所有相关切块，按相似度聚类，每个簇代表一个考点，用户可选择指定簇内集中出题

### Requirement: 跨 PDF 聚类综合组卷
系统 SHALL（未来阶段）支持定时任务对指定分类/合集内所有 content 切块做 KMeans 或 HDBSCAN 向量聚类，自动划分知识点簇，支持配置难度配比（默认 易:中:难 = 6:3:1），从不同聚类不同 PDF 中均匀抽源块出题，避免考点集中在单篇文档。

#### Scenario: 自动组卷
- **WHEN** 用户请求综合组卷（整套试卷）
- **THEN** 系统 SHALL 从多个聚类簇中按难度配比抽取源块，确保题目覆盖多个知识点且来源分散于多篇文档

### Requirement: 聚类任务异步化
聚类任务 SHALL 设计为异步后台任务，不阻塞出题接口响应。

#### Scenario: 聚类完成后出题
- **WHEN** 后台聚类任务完成
- **THEN** 前端可感知聚类状态（通过轮询或 WebSocket），完成后允许基于聚类结果组卷出题

> **注意**：quiz-clustering 明确标记为未来阶段，本次变更仅做数据模型和 API 设计预留，不做具体实现。
