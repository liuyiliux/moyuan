## ADDED Requirements

### Requirement: 手动本地备份
系统 SHALL 支持用户通过 UI 一键触发手动备份，备份内容包括 PostgreSQL 数据库导出（pg_dump）和原始文件目录打包，备份文件存储到用户指定路径。

#### Scenario: 执行手动备份
- **WHEN** 用户在「数据管理」页面点击「创建备份」
- **THEN** 系统生成包含数据库转储和文件压缩包的备份文件，显示备份进度，完成后提示备份文件路径

#### Scenario: 备份文件命名
- **WHEN** 备份完成
- **THEN** 备份文件以时间戳命名（如 `mkb-backup-2026-05-29.tar.gz`），包含版本信息便于识别

### Requirement: 完整知识库导出
系统 SHALL 支持将完整知识库（数据库 + 文件 + 配置）导出为单一压缩包，用于迁移或完整备份。

#### Scenario: 导出完整知识库
- **WHEN** 用户选择「导出知识库」
- **THEN** 系统打包数据库数据、所有原始文件和系统配置（不含 API keys），生成标准化导出包

### Requirement: 知识库导入与恢复
系统 SHALL 支持从标准导出包恢复知识库，导入时自动处理路径重映射和数据库恢复。

#### Scenario: 从备份恢复知识库
- **WHEN** 用户选择「从备份恢复」并选择备份文件
- **THEN** 系统执行数据库恢复（pg_restore）和文件解压，根据新环境路径重新映射文件引用，恢复完成后展示恢复摘要

#### Scenario: 跨机器迁移
- **WHEN** 用户在新机器上导入从旧机器导出的知识库包
- **THEN** 系统自动适配新机器的文件存储路径，所有内容可正常访问和检索
