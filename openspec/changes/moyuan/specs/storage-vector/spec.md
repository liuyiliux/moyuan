## ADDED Requirements

### Requirement: 统一数据库存储业务数据与向量
系统 SHALL 使用 PostgreSQL 16 + pgvector 作为唯一数据库，业务表与向量表共存于同一数据库实例中，通过外键关联。

#### Scenario: 业务数据与向量联合查询
- **WHEN** 执行语义检索时
- **THEN** 系统在单次数据库查询中完成向量相似度计算与业务字段过滤，无需跨系统调用

### Requirement: 多模态向量嵌入存储
系统 SHALL 为每个内容单元存储其对应的向量嵌入，文本内容使用文本嵌入模型，图片使用多模态/图像嵌入模型，视频字幕切片使用文本嵌入模型。

#### Scenario: 文本内容嵌入存储
- **WHEN** 一段文本内容完成解析
- **THEN** 系统调用配置的文本嵌入 API 生成向量，存储到 `content_embeddings` 表，记录模型名称与维度

#### Scenario: 图片内容嵌入存储
- **WHEN** 图片 OCR 与内容理解完成
- **THEN** 系统调用图像嵌入 API 生成图像向量，与文本描述向量分别存储，供不同检索策略使用

#### Scenario: 嵌入模型切换处理
- **WHEN** 用户在设置中切换嵌入模型
- **THEN** 系统提示「旧嵌入向量与新模型不兼容，建议重新索引」，并提供批量重新嵌入操作

### Requirement: 本地文件存储路径管理
系统 SHALL 将原始文件存储在本地文件系统中，路径由用户在设置中配置，数据库中存储文件的相对路径。

#### Scenario: 配置自定义存储路径
- **WHEN** 用户在设置中指定文件存储目录
- **THEN** 后续上传的所有文件存储至该目录下，按年月日层级自动分类存放

#### Scenario: 存储路径迁移
- **WHEN** 用户修改存储路径后执行「迁移文件」操作
- **THEN** 系统将现有文件复制到新路径，更新数据库中的路径记录，迁移完成后原路径文件保留直到用户手动删除

### Requirement: Alembic 数据库版本管理
系统 SHALL 使用 Alembic 管理数据库 schema 迁移，所有表结构变更通过迁移脚本版本化管理。

#### Scenario: 初始化数据库
- **WHEN** 首次运行系统时执行初始化
- **THEN** 系统运行 Alembic 迁移脚本，创建所有必要的表和索引，包括 pgvector 扩展的 IVFFlat 索引

### Requirement: 嵌入处理失败重试
系统 SHALL 在嵌入 API 调用失败时标记内容状态为「嵌入失败」，支持用户手动触发重试，内容原始数据不受影响。

#### Scenario: 嵌入 API 临时不可用
- **WHEN** 调用嵌入 API 时网络超时
- **THEN** 内容记录被标记为「嵌入待重试」状态，用户可查看失败列表并批量重试，已存储的文本内容不受影响
